# ChatApp - Онлайн чат на C#

## Структура проекта

- **ChatApp.Server** - ASP.NET Core Web API с SignalR для real-time коммуникации
- **ChatApp.Client** - WPF приложение (Windows Desktop)
- **ChatApp.Shared** - Общие модели и DTO

## Функционал

### Аутентификация
- ✅ Регистрация и вход по email/паролю
- ✅ JWT токены для авторизации
- ✅ Защита от дубликатов email
- ✅ Вход с одного аккаунта на разных компьютерах

### Профиль пользователя
- ✅ Загрузка и отображение аватара
- ✅ Редактирование имени пользователя
- ✅ Статус (Онлайн/Офлайн/Не беспокоить)
- ✅ Биография (до 500 символов)
- ✅ Настройки уведомлений
- ✅ Смена пароля
- ✅ Выход из аккаунта

### Чат
- ✅ Личные диалоги в реальном времени
- ✅ Отправка текстовых сообщений
- ✅ Прикрепление файлов (изображения, документы до 10MB)
- ✅ Загрузка истории сообщений
- ✅ Поиск по контактам
- ✅ Отображение статуса пользователей
- ✅ Статусы доставки (отправлено/доставлено/прочитано)
- ✅ Отображение изображений в чате
- ✅ Визуальное различие своих и чужих сообщений

### Уведомления
- Real-time через SignalR
- Обновление статусов сообщений
- Изменение статуса пользователей

## Запуск проекта

### 0. Установка необходимых компонентов

Если .NET SDK не установлен, скачайте и установите:
https://dotnet.microsoft.com/download/dotnet/8.0

После установки .NET SDK, установите EF Core tools:

```
dotnet tool install --global dotnet-ef
```

### 1. Настройка базы данных (SQLite)

База данных уже создана! Файл `chatapp.db` находится в папке ChatApp.Server.

Если нужно пересоздать базу:

```
cd ChatApp.Server
dotnet ef database update
```

### 2. Запуск сервера

Сервер уже запущен! Работает на http://localhost:56188

**Если нужно перезапустить:**

Способ 1 (простой):
```
restart-server.bat
```
Этот файл автоматически остановит старый процесс и запустит новый.

Способ 2 (вручную):
```
cd ChatApp.Server
dotnet run
```

**Если порт занят (ошибка "address already in use"):**

Способ 1: Запустите `stop-server.bat` чтобы остановить старый процесс

Способ 2: Найдите и закройте окно с запущенным сервером

Способ 3: Смотрите подробную инструкцию в `КАК_ОСТАНОВИТЬ_СЕРВЕР.txt`

### 3. Запуск клиента

**Способ 1: Через Visual Studio**

1. Откройте файл `ChatApp.sln` в Visual Studio (двойной клик)
2. В Solution Explorer найдите проект `ChatApp.Client`
3. Правой кнопкой мыши → Set as Startup Project
4. Нажмите F5 или кнопку "Start" (зеленая стрелка)

**Способ 2: Через командную строку**

```
cd ChatApp.Client
dotnet run
```

### 4. Использование приложения

1. При запуске откроется окно входа
2. Нажмите "Регистрация" для создания аккаунта
3. Введите email и пароль
4. После регистрации войдите с этими данными
5. Откроется главное окно чата

## 🌐 Подключение с другого компьютера

Хотите подключиться к серверу с другого компьютера в локальной сети?

**Быстрое решение:**
1. На компьютере с сервером: `ipconfig` → запишите IPv4 адрес
2. На компьютере с клиентом: откройте `ChatApp.Client\appsettings.json`
3. Измените `"ServerUrl": "http://localhost:56188"` на `"ServerUrl": "http://192.168.1.100:56188"` (ваш IP)
4. Пересоберите клиент: `dotnet build`

**Вход с одного аккаунта на разных компьютерах:**
- ✅ Можно входить с одного email/пароля на нескольких компьютерах одновременно
- ✅ Система автоматически запрещает создание дубликатов email

## 📧 Настройка Email (опционально)

Для отправки кодов восстановления пароля на email:

1. Откройте `ChatApp.Server/appsettings.json`
2. Замените placeholder значения в секции `EmailSettings`
3. Следуйте инструкциям в [EMAIL_SETUP.md](EMAIL_SETUP.md)

**Без настройки email:**
- Приложение работает в демо-режиме
- Код восстановления показывается на экране
- Все остальные функции работают нормально
